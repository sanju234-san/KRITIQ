# Project Rules & Customizations

* **Git / GitHub Rule**: Always ask the user for explicit confirmation before pushing any changes to the remote repository on GitHub (`git push`).

## Folder Boundaries & Coding Constraints
* **Folder Assignment**: Working as **Sayeed (Backend & Database)**.
  * Your assigned folder is **`kritiq-backend/app/`**. Do not modify any files outside this folder (e.g., `kritiq-backend/ai_agent/`, `kritiq-backend/mcp_server/`, `rag_pipeline/`, `repo_integration/`, `dataset/`, `cli/`, `kritiq-frontend/`) unless explicitly instructed.
  * Do not move files or rename folders.
* **Shared Files**: Do not modify shared files unless explicitly instructed.
* **Module Imports**: Import other modules only through their public interfaces. If an interface is missing, leave a `TODO` instead of implementing another member's work.
* **Coding Protocol**:
  * **Before writing code**: Explain implementation plan, list every file modified, and wait for approval if the scope expands outside `kritiq-backend/app/`.
  * **After coding**: Show all modified code. Do NOT run or test anything automatically.
  * **Playgrounds**: Create any playground/test scripts inside `kritiq-backend/app/` only, adhering to structured PASS/FAIL reporting.
